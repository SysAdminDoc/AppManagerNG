## Guiding Principles

AppManagerNG is a "friendlier front door" to App Manager's engine. Three commitments shape every prioritization call:

1. **Progressive disclosure, not feature removal.** Power-user depth stays; it just doesn't ambush newcomers. The Pro Mode toggle is the primary mechanism.
2. **Upstream as reference, not competitor.** Pull fixes and features from upstream aggressively. Diverge only where UX philosophy demands it.
3. **Rootless first.** Shizuku-based operations are first-class citizens alongside root. The majority of Android users have no root; they shouldn't hit a dead end.

Hard constraints:
- License: **GPL-3.0-or-later** throughout. Every new dependency must be compatible.
- minSdk **21** (Android 5.0). Many modern APIs need compat paths or conditional activation.
- No AI-generated code contributions (inheriting upstream policy; NG will publish its own CONTRIBUTING.md in v0.2.0).

## Tier Definitions

| Tier | Meaning |
|------|---------|
| **Now** | Tied to v0.2.0–v0.3.0 or is a critical bug/regression fix |
| **Next** | Target v0.4.0–v0.6.0 horizon; high-value, well-scoped |
| **Later** | Post-v0.6.0; valuable but requires upstream landing or prior work |
| **Under Consideration** | Not rejected — needs architecture decision or external dependency |
| **Rejected** | Explicitly out of scope; reasoning provided |

## Iter-23 Repo Reconciliation + 2026 Platform Delta (2026-05-16)

Phase-0 state memo: AppManagerNG is an Android power-user package manager fork, not a generic app store or desktop debloater. The current repo is Java/Kotlin Android Views + Material Components, Gradle/AGP, GPL-3.0-or-later, minSdk 21, targetSdk 36, release 0.4.2 / versionCode 6. Its stated philosophy is "retain upstream App Manager power, remove friction": progressive disclosure, rootless/Shizuku/ADB first, no Compose migration yet, no telemetry by default, and no cloud/account dependency for core workflows. Local reconnaissance found the shipped source largely consistent with that philosophy, but several roadmap/docs claims have drifted from disk: `README.md` and the old roadmap claim an NG-specific `CONTRIBUTING.md` exists when it does not; release docs advertise ABI choices while `release.yml` uploads only the universal APK; lint/tests workflows do not trigger for the actual `main` branch; and the changelog claims dependency-check `13.1.1` while `versions.gradle` pins `10.0.3`.

External signal snapshot (GitHub/API snapshot 2026-05-16): upstream AM remains the capability reference (8,025 stars, 213 open issues, pushed 2026-04-16 [S291]); Canta (4,780 stars, pushed 2026-05-12 [S292]), UAD-NG (6,606 stars, pushed 2026-05-11 [S293]), Hail (5,821 stars, pushed 2026-04-26 [S294]), Inure (1,757 stars, pushed 2026-05-16 [S295]), SD Maid SE (6,740 stars, pushed 2026-05-16 [S296]), Neo Backup (3,614 stars, pushed 2026-05-03 [S297]), Android-DataBackup (6,690 stars, pushed 2026-05-14 [S298]), Shizuku (25,130 stars [S299]), LibChecker (6,866 stars, pushed 2026-05-16 [S300]), AppVerifier (1,046 stars [S301]), and Obtainium (17,115 stars [S302]) define the active OSS feature envelope. Commercial signal says users still pay for polished backup scheduling/cloud destinations (Swift Backup, Titanium Backup [S305], [S306]), app history/tags/insights (AppDash [S304]), and lower-friction desktop debloat flows (ADB AppControl [S303]). NG should harvest the workflow lessons, not the account-bound or desktop-only architecture.

## Upstream Sync Strategy

AppManagerNG is not a hard fork. The intent is to stay semantically close to upstream while diverging on UX.

**Pull policy:**
- Security fixes from upstream: pull immediately regardless of conflict cost.
- Bug fixes: cherry-pick within one upstream release cycle.
- Upstream "Upcoming Features" (Finder, APK editing, Routine Ops, Crash Monitor, Database Viewer, Terminal): pull when upstream ships rather than reimplement. NG reimplements only if upstream stalls for >12 months.

**Divergence tracking:**
- Weekly CI check comparing NG `main` against upstream `master`; generates cherry-pick candidate list.
- Maintained in `.github/upstream-sync.md` (to be created in v0.2.0).
