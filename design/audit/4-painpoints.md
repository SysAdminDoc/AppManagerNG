# Phase 4 Pain-Point Inventory

This inventory lists dated or cheap-looking surfaces and the concrete facelift direction for each. The fixes are visual and structural only; they must not change feature behavior, intent filters, AppWidgets, or root/ADB operations.

| Surface | Current state | Proposed fix | Files to change | Reference |
| --- | --- | --- | --- | --- |
| Title bar | `activity_main.xml`, Settings, App Details, and most activities use a generic Material toolbar at actionBar height. Main search is injected as an ActionBar custom view, so the search experience is central in code but not in XML. | Use a v2 toolbar with 64dp height, stronger title/search field hierarchy, branded focus ring, and scrolled surface tint. For MainActivity, keep the existing ActionBar search binding but wrap the toolbar visually as a search-first command bar. | `activity_main.xml`, `activity_settings.xml`, `activity_settings_dual_pane.xml`, `activity_app_details.xml`, `libcore/ui/src/main/res/values/styles.xml`, new v2 toolbar styles. | Linear command bar, Arc sidebar/search chrome. |
| Filter chip row | Quick filters exist and are useful, but every chip has near-identical weight. Sort, filters, active toggles, and clear action compete visually. | Split chip roles: sort as tonal suggestion chip with sort icon, filter chips as outlined inactive and filled tonal active, clear as low-danger utility. Preserve IDs and checked behavior. | `activity_main.xml`, `Widget.AppTheme.Chip.MainFilter`, `Widget.AppTheme.Chip.MainSuggestion`, `MainActivity.bindQuickFilterChips()` only if dynamic tinting is needed. | Things 3 filter chips, Linear filter menus. |
| Aggregate banner | `list_status` is a text-only status line: "Showing N of M apps - trackers - dangerous perms". It is useful but visually low-signal on dark surfaces. | Convert to a compact metric strip: total apps, tracker count, dangerous permission count as colored count chips with small icons. Keep `list_status` ID for binding; if richer children are needed later, the TextView remains the accessibility summary and click target. | `activity_main.xml`, `main_status_background.xml`, possible new metric drawables, `MainActivity.updateMainListState()` for spans or compound drawables. | 1Password vault summaries, Linear status pills. |
| App row tracker badges | `tracker_indicator` is a number-only chip with adapter-set text color. Users must know what the number means. | Semantic thresholds: green <= 4, amber 5-19, red >= 20; all-blocked uses success tint and filled shield/check icon. Preserve `tracker_indicator` ID and click-through to tracker tab. | `item_main.xml`, `layout-w600dp/item_main.xml`, `MainRecyclerAdapter` color threshold logic, badge drawables/selectors. | 1Password risk badges, Obsidian metadata tags. |
| App row dangerous-perms badge | `perm_indicator` uses the same badge background and text color model as trackers; ratio meaning is not visually encoded. | Tint by granted/total ratio: success 0 granted, warning below 50 percent, danger 50 percent or higher. Keep `perm_indicator` ID and App Details permissions click-through. | `item_main.xml`, `layout-w600dp/item_main.xml`, `MainRecyclerAdapter` badge color logic. | Linear issue severity tags. |
| Empty filter result | `view_main_empty_state.xml` is a functional stock empty state with title, summary, and action. The no-results state is better than before but still generic. | Add a small package/filter illustration, explicit active filter names, and a reset-filter affordance styled as primary tonal. Keep `empty_state_title`, `empty_state_summary`, and `empty_state_action`. | `view_main_empty_state.xml`, `MainActivity.describeActiveQuickFilters()`, empty-state styles/drawables. | Notion empty-state illustration. |
| Dialogs | Many dialogs use default MaterialAlertDialog content with dense forms. High-risk operations rely on text, not surface tone or action hierarchy. | Use tonally tinted dialog containers, semantic icons, sticky bottom action bar, clear destructive button treatment, and better content grouping. Keep button labels and existing dialog flows. | All `dialog_*.xml`, `MaterialAlertDialogBuilder` call sites, `AppTheme.FullScreenDialog.Animation`, dialog styles. | 1Password confirmation dialogs, Notion modal tonality. |
| Settings | Single-pane settings are plain; dual-pane structure is good but visually utilitarian. Category hierarchy and entry icons are not strong enough. | Use tiered typography, section dividers, entry icons, selected pane state, and a stronger Appearance section for theme preview. Preserve dual-pane width behavior. | `activity_settings.xml`, `activity_settings_dual_pane.xml`, `m3_preference*.xml`, preference styles, `SettingsActivity`. | Arc settings sidebar, 1Password preferences. |
| Loading | Loading mostly uses indeterminate `LinearProgressIndicator` or circular progress. Lists jump from blank/progress to content. | Add skeleton rows with shimmer-like alpha animation using platform drawables/animators only. Respect transition animation scale and reduced-motion helper. | `activity_main.xml`, list layouts, new skeleton row/layout drawables, progress styles. | Linear loading rows, Notion skeletons. |
| Pro Mode toggle | Pro Mode is currently a settings-style control. It is strategically important but visually undersold. | Add a hero card in About/Onboarding with feature comparison and CTA. Settings keeps the actual toggle; the hero drives discovery only. | `fragment_onboarding.xml`, About/help surface, settings appearance screen, strings added with xliff conventions. | 1Password onboarding trust cards. |
| App icon / launcher brand | Current icon remains upstream-derived assets plus documented logo prompts. Brand notes prefer package/cube with gear/key/shield and green-to-cyan. | Commit to the app-icon/emblem direction, refresh adaptive icon foreground/background, round icon, monochrome themed icon, README logo, fastlane icon. Do this only after licensing/source files are ready. | `branding/logo-prompts.md`, `docs/raw/images/icon.png`, `app/src/main/res/drawable/ic_launcher_foreground.xml`, `mipmap-*`, monochrome icon resources, fastlane metadata. | Arc app icon confidence, Material themed icon guidance. |
| Main list row | `item_main.xml` is a large card containing app icon, label, package, badges, date, UID, version, type, SDK, signature, issuer, and backup metadata. | Keep the same data but give it lanes: identity lane, risk badge lane, technical metadata lane. Use quieter card stroke, stronger app label, less equal-weight text, and semantic badge tint. | `item_main.xml`, `layout-w600dp/item_main.xml`, `MainRecyclerAdapter` only for tint thresholds. | 1Password vault list, Bear typography. |
| App Details header | `activity_app_details.xml` is toolbar + tabs + pager. App identity is not a first-viewport anchor at the activity level. | Add a premium app detail header inside the pager stack: icon, label, package/version, trust/risk chips, primary actions. Keep all tabs and pager behavior. | `activity_app_details.xml`, `pager_app_info.xml`, `item_app_info_action.xml`, App Details fragments. | 1Password item detail, Obsidian properties. |
| App Usage rows | `item_app_usage.xml` packs percent, date, screen time, data, Wi-Fi, and progress into a small card. | Make usage percentage a quiet leading metric, align data columns, and use semantic progress tint. Keep all IDs and adapter data unchanged. | `item_app_usage.xml`, `activity_app_usage.xml`, `AppUsageAdapter`. | 1Password list metrics, Things spacing. |
| Debloater rows | `item_debloater.xml` uses an outlined card with list type and description. The safety model is not visually prominent. | Add safety-rating color bands and confidence chips. Destructive actions get danger treatment; safe/recommended gets success treatment. | `item_debloater.xml`, `dialog_bloatware_details.xml`, debloater adapter color mapping. | Canta/UAD-NG safety ratings, Notion callouts. |
| Interceptor | `activity_interceptor.xml` is a heavy multi-section screen with many RecyclerViews. It is powerful but visually stressful. | Use grouped panels with sticky action hierarchy, better section headings, and clear source/target intent affordances. Preserve every intent handling path. | `activity_interceptor.xml`, `ActivityInterceptor.java`, related row layouts. | Arc command panels, Obsidian power-user UI. |
| File Manager | `activity_fm.xml` and `fragment_fm.xml` are functional but closer to system utility UI than premium tool UI. | Add stronger path bar, breadcrumb hierarchy, denser but clearer rows, and semantic icons for symlink/root/permission states. | `activity_fm.xml`, `fragment_fm.xml`, `item_fm.xml`, `item_path.xml`, file-manager drawables. | Obsidian file sidebar, Arc spatial UI. |
| AppWidgets | AppWidget layouts have separate constraints and are currently styled as utility widgets. | Preserve RemoteViews compatibility. Use the v2 palette, system widget radii, clearer metric typography, and no unsupported Material widgets. | `app_widget_*.xml`, `widget_recording.xml`, `values-v31/styles.xml`, `values-night-v31/styles.xml`. | Android 12+ system widgets, 1Password lock-screen style density. |

## Priority Order

1. MainActivity shell and `item_main.xml`.
2. Semantic tracker/dangerous-permission badges.
3. Settings single/dual-pane.
4. App Details header.
5. Filter/sort sheet.
6. Empty/loading states.
7. Dialogs for destructive operations.
8. Debloater and App Usage.
9. File Manager, Logcat, Finder, Interceptor.
10. AppWidgets and final icon refresh.

## Non-Negotiables

- Preserve all IDs bound by Java adapters and activities.
- Keep existing strings and xliff placeholder conventions.
- No Compose.
- No new third-party dependencies unless separately justified and REUSE-audited.
- Every semantic color must pass WCAG 2.2 AA against its background.
- Every destructive operation keeps explicit confirmation and accessible button labels.
